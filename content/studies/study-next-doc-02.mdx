#### 라우트 생성

Next.js는 **폴더**를 사용하여 경로를 정의하는 파일 시스템 기반 라우터를 사용한다.

각 폴더는 **URL** 세그먼트에 매핑되는 [route segment](https://nextjs.org/docs/app/building-your-application/routing#route-segments)를 나타낸다. [중첩된 경로](https://nextjs.org/docs/app/building-your-application/routing#nested-routes)를 만들려면 폴더를 서로 중첩하면 된다.
![image](https://github.com/tamoimi/tami-dev-blog/assets/100749520/3cb02195-aa4e-4c0d-9f97-69494730c385)
특별한 [page.js file](https://nextjs.org/docs/app/building-your-application/routing/pages-and-layouts#pages)은 라우트 세그먼트에 공개적으로 액세스할 수 있도록 하는 데 사용된다.
![image](https://github.com/tamoimi/tami-dev-blog/assets/100749520/d49143dd-f812-44df-9955-df83d476a4c7)
위 예시에서 `/dashboard/analytics` URL 경로에는 해당 `page.js` 파일이 없기 때문에 공개적으로 액세스할 수 없다. 이 폴더는 컴포넌트, 스타일시트, 이미지 또는 기타 같은 위치에 있는 파일을 저장하는 데 사용될 수 있다.

☄️ `.js`, `.jsx` 또는 `.tsx` 파일 확장자를 특수 파일에 사용할 수 있다.

---

#### UI 생성

각 경로 세그먼트에 대한 UI를 생성하는 데 [특수 파일 규칙](https://nextjs.org/docs/app/building-your-application/routing#file-conventions)이 사용된다. 가장 일반적인 것은 경로에 고유한 UI를 표시하는 [페이지](https://nextjs.org/docs/app/building-your-application/routing/pages-and-layouts#pages)와 여러 경로에서 공유되는 UI를 표시하는 [레이아웃](https://nextjs.org/docs/app/building-your-application/routing/pages-and-layouts#layouts)이다.

예를 들어 첫 번째 페이지를 만들려면 `app` 디렉터리 내에 `page.js` 파일을 추가하고 React 구성 요소를 내보낸다:

```tsx
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```
