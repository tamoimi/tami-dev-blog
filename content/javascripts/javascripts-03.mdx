#### 브라우저 렌더링 과정

웹 페이지의 브라우저 렌더링 과정은 여러 단계로 이루어지며, 각 단계는 웹 페이지를 화면에 표시하기 위해 필요한 작업을 수행합니다. 대략적으로 다음과 같은 과정을 거칩니다:

1. **문서 파싱(Document Parsing)**
   - **HTML 파싱**: 브라우저는 서버로부터 받은 HTML 문서를 읽기 시작합니다. 이 과정에서 브라우저는 HTML을 DOM(Document Object Model) 트리로 변환합니다. DOM 트리는 문서의 구조를 메모리 내에 객체로 표현한 것입니다.
   - **CSS 파싱**: HTML과 함께 또는 HTML 내에 포함된 CSS는 CSSOM(CSS Object Model) 트리로 변환됩니다. CSSOM 트리는 스타일 정보를 담고 있어, 각 DOM 요소가 어떻게 보여야 할지 결정합니다.
2. **렌더 트리 구축(Render Tree Construction)**
   - DOM 트리와 CSSOM 트리를 결합하여 렌더 트리를 생성합니다. 렌더 트리는 화면에 표시되는 요소들만을 포함하며, 시각적 속성(크기, 색상 등)을 포함합니다. 화면에 보이지 않는 요소들(**`display: none;`** 등)은 렌더 트리에 포함되지 않습니다.
3. **레이아웃(Layout)**
   - 렌더 트리가 완성되면, 각 노드의 정확한 위치와 크기를 계산하는 레이아웃(또는 리플로우) 과정이 진행됩니다. 이 단계에서는 페이지의 각 요소가 화면의 어디에 위치해야 하는지 결정합니다.
4. **페인팅(Painting)**
   - 레이아웃 단계 후, 실제 요소들을 화면에 그리는 페인팅 과정이 이루어집니다. 이 과정에서는 레이아웃 단계에서 계산된 정보를 바탕으로 각 요소의 색상, 이미지, 텍스트 등을 화면에 그립니다.
5. **합성(Compositing)**
   - 복잡한 페이지에서는 여러 레이어로 나누어 처리할 수 있습니다. 각 레이어는 독립적으로 페인트 될 수 있고, 최종적으로는 이러한 레이어들이 합쳐져서 최종 이미지를 생성합니다. 이 과정은 페이지의 일부분만 변경되어도 전체 페이지를 다시 그릴 필요가 없게 해줌으로써 성능을 향상시킵니다.

각 단계는 선형적으로 진행되는 것처럼 보이지만, 실제로는 사용자와의 상호작용이나 자바스크립트의 실행 등으로 인해 과정이 반복될 수 있습니다. 예를 들어, 자바스크립트를 통해 DOM을 변경하면, 브라우저는 레이아웃과 페인팅 단계를 다시 거치게 됩니다. 최적화된 웹 페이지를 만들기 위해서는 이러한 렌더링 과정을 이해하고, 필요 없는 레이아웃 재계산과 페인팅을 최소화하는 것이 중요합니다.
